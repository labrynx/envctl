## Documentation model

This repository uses four documentation types:

- **Getting started** → learn fast
- **Concepts** → understand the model
- **Workflows** → solve real problems
- **Reference** → look up exact behavior
- **Internals** → contribute safely

This separation is intentional.

It keeps onboarding, usage, lookup, and contribution material from collapsing into one giant document.
